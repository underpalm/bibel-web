# Rose Landing Page Design Guidelines

This local landing page keeps the existing visual language while removing commerce behavior.

## Visual Tokens

- Primary green: `#17873f`
- Interaction green: `#0aa34a`
- Ink: `#000000`
- Muted text: `#666666`
- Soft border: `#e5e7eb`
- Backgrounds: white and light neutral `#f7f7f7`
- Radius: pill controls use full radius, framed surfaces stay at `8px` or less
- Lines: thin 1px dividers structure sections without decorative cards

## Type

- Sans-serif, light editorial headings.
- Large section headings stay calm, spacious, and direct.
- Body copy uses short line lengths and clear hierarchy.
- Letter spacing remains `0` for reliable rendering.

## Layout

- Fixed header with centered wordmark and restrained navigation.
- Full-viewport media hero with text and CTA over the image.
- Landing sections are full-width bands or split layouts.
- The AI conversation preview is a functional framed tool surface.
- Footer is split into app summary, updates form, grouped links, and copyright.

## Interaction

- Mobile navigation uses a drawer.
- Hero rotates through three image-led statements.
- Demo prompts update the AI conversation preview.
- Voice button shows a listening pulse.
- Waitlist and update forms give inline confirmation.

## Asset Handling

The landing page intentionally keeps local copies of the existing image set and the same font stack for visual continuity. Replace them with owned app assets before public release.
